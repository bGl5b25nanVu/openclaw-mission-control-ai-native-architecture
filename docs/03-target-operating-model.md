# 03 Target Operating Model

## Top-level model
1. Goals define desired outcomes
2. Domains group related loops around those goals
3. Loops operate continuously against signals
4. Runs execute specific attempts
5. Exceptions escalate to humans only when needed
6. Evidence determines whether a run improved the system

## Human role
Humans should mainly handle:
- strategic prioritization
- policy decisions
- exception handling
- external-risk approvals
- resource reallocation

## AI role
AI should mainly handle:
- observing signals
- proposing actions
- executing within policy
- collecting evidence
- reviewing outcomes
- escalating only when blocked or high-risk

## Operating rhythm
Default rhythm should shift from task tracking to loop management:
- active loops
- stalled loops
- failed loops
- loops requiring policy change
- loops requiring human intervention
