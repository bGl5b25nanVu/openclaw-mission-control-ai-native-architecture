# 07 Loop Semantics for Existing Mission Control Boards

## Purpose
Preserve the current Mission Control object model while changing the meaning of key boards from static work surfaces into continuous operating loops.

## Reframing table

|---|---|
| 年度KPI主盘 | Goal Governance Loop |
| 经营复盘与纠偏 | Review & Correction Loop |
| 本周经营优先级 | Weekly Prioritization Loop |

| 内容与分发 | Content Growth Loop |

| Dashboard模块规格 | Control Plane Definition Loop |
| Dashboard首版MVP | Control Plane Delivery Loop |

## Rules for loop semantics
Each loop should define:
- signal inputs
- target condition
- default run cadence
- allowed actions
- evidence required
- escalation conditions
- human intervention boundary

## Practical migration strategy
1. Keep existing board records
2. Rewrite objectives and success metrics around loops
3. Rewrite tasks as runs / run contracts
4. Add exception-aware dashboard views later
